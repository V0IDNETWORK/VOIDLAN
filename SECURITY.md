# Security Policy

## Supported Versions

Security updates are provided for the latest stable release.

| Version | Supported |
|---------|-----------|
| Latest  | ✅ |
| Older versions | ❌ |

## Reporting a Vulnerability

Please do not open a public GitHub issue for security vulnerabilities.

Instead, report security issues privately through:

- GitHub Security Advisories
- Private contact with the maintainer

Include:

- Description of the vulnerability
- Steps to reproduce
- Possible impact
- Suggested fix (if available)

## Response Process

We will:

1. Confirm receipt of the report.
2. Investigate the issue.
3. Prepare a fix if necessary.
4. Publish a security update.

## Security Principles

VOID LAN follows these principles:

- No unnecessary data collection
- Local-first architecture
- Minimal network exposure
- Secure file transfer practices
- Responsible disclosure
